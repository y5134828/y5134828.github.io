记录自己搭建tp技术博客(一)


  最近在学习一些thinphp框架的知识，跟着写了一个个人博客，虽然没什么高大上的技术成分，也是自己的一些学习经历，就在这里做个记录，写一些自己的学习感悟也把一些问题记录下来，一边记录一边成长。整体项目通过thinkphp框架搭建，从开始安装框架到基本完成功能花了一个月时间，又购买服务器，备案，购买域名，上线调试又花了一个月时间。也可能是自己没用心写的原因，才花了这么久。放一下链接~~p先生的个人网站

1.框架的安装
  从thinkphp的官网或者github都可以下载完整的框架项目。这里引用一下开发文档的目录结构，我感觉这个写的很清楚明了。

│ │ ├─model 模型目录

2.thinkphp的使用
  这里牵扯到一些搭建php的环境问题，需要安装apache，mysql数据库等。网上这方面的教程也比较多，当然也可以使用一些集成环境，例如phpstudy，wamp等工具。
 环境安装好以后一般会在apache目录下生成www文件夹，可以在www文件夹中创建一个空的项目文件夹，将下载好的tp框架放到你创建的项目文件夹中，可以通过localhost：项目名访问，来测试tp框架是否正常运行。

成功显示的图片

3.当成功安装好框架后，可以在config.php应用（公共）配置文件中开启debug调试，trace等，方便代码编写时候的调试。thinkphp框架采用单一入口的方式。入口文件为public目录下index.php，当然也可以把入口文件放到根目录下（在线上采用虚拟机时这一块需要修改），正常情况放在public目录下相对安全。
4.thinkphp框架
    thinkphp框架是基于mvc的一个框架，首先应该先要理解mvc的框架思想。mvc框架：软件框架是项目软件开发过程中提取特定领域软件的共性部分形成的体系结构，不同领域的软件项目有着不同的框架类型。框架不是现成可用的应用系统。而是一个半成品，提供了诸多服务，开发人员进行二次开发，实现具体功能的应用系统。~~~还有一种mvc的设计模式：是一套被反复使用、多数人知晓的、经过分类编目的、代码设计经验的总结，它强调的是一个设计问题的解决方法。
    使用 MVC（Model View Controller 模型-视图-控制器）设计创建 Web 应用程序，Model（模型）是应用程序中用于处理应用程序数据逻辑的部分。通常模型对象负责在数据库中存取数据；View（视图）是应用程序中处理数据显示的部分。通常视图是依据模型数据创建的；Controller（控制器）是应用程序中处理用户交互的部分。通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据。
